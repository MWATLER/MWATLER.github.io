Namespaces

One of the first steps to object oriented programming is the namespace. The namespace is present in C++ but not in C. Object oriented programming leads to modular programming, and there are cases where classes, structures, functions or variables from different modules may have to be differentiated from each other. For example, you might be writing some code that has a function called xyz() and there is another library or module available which also has a function called xyz(). Now the compiler has no way of knowing which version of the xyz() function you are referring to within your code.

A namespace is designed to overcome this difficulty and is used as additional information to differentiate similar functions, classes, variables etc. with the same name available in different libraries. Using namespace, you can define the context in which names are defined. In essence, a namespace defines a scope. See Namespaces under Sample Code for a simple example.

Object Oriented Programming

We have looked at many examples where we group variables that are related to each other into structures. You may have noticed that there are functions in this sample code that act on the data in these structures. For example, the income tax program from week 6 collected personal data into the structure Person. There were functions that operated on the data in this structure to calculate the deductible and to calculate the income tax:
void CalculateDeductible(struct Person* person);
void CalculateIncomeTax(struct Person* person);
The code from week 6 can be found at IncomeTax.h and IncomeTax.c.

What if we collected variables that are related to each other, and functions that operate on those variables, together. For instance, what if we collected the structure Person with the functions CalculateDeductible() and CalculateIncomeTax! Related variables, and functions which act on those variables, can be collected into something called a class. These functions have access to all the variables of the class and these variables no longer have to be passed through as an argument to these functions. See Income Tax under Sample Code for a code example.

A class is therefore a collection of related variables along with functions that operate on them. A class is a blueprint, or a user defined type, similar to a structure. An instance of a class is an object. Please see ObjectOrientedProgramming.pptx for an introduction to object oriented programming. See the Travel Simulator under Sample Code for the code for the travel simulator introduced in the powerpoint document.
