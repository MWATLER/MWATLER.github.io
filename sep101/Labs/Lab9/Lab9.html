Card Game - Twenty One

You will write a program that simulates a game of 21 with a few twists. The purpose of 21 is to draw cards that sum as closely to 21 without going over. The player with the highest sum without going over 21 wins. In this game however, the players will have the option to steal the cards of other players.

The player has a name, an unknown number of cards with values 1-10 each, and cash. The player will have to allocate the cards once the number of cards is known.

The player creates itself given his/her name.
The player draws cards by deleting the cards it has and creating a new list of cards based on the number of cards requested. The value of each card is according to a random number generator that generates a random number from 1 to 10: (rand()%10)+1.
The player sums cards by going through the list of cards it has and returning the total points for all of them.
The player can add cash given an amount. This amount could be negative (the player lost). If the player has no cash left then add cash returns a false, otherwise it returns a true.
The player can steal cards from another player. In this case the player has to delete its own cards, copy the cards from the other player, then delete the other player's cards. YOU HAVE TO ADD A TEST SO A PLAYER DOESN'T STEAL CARDS FROM ITSELF.
The player can generate a report printing the player's name and cash left over.
The player can destroy itself by cleaning up any memory allocated.

The file PlayerMain.cpp has been given to test your code.

Sample code on copying, moving and stealing data from one object to another can be seen in an example with a BankAccount class, in the following code:
BankAccount.h, BankAccount.cpp and BankAccountMain.cpp.
