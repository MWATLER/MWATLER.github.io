Structures

In C programming, a struct (or structure) is a collection of variables (can be of different types) under a single name. These variables are related to each other and are therefore grouped in a structure. Just as we have predefined types such as int's and float's in C, a structure is a user defined data type.
Let us say we need to store the data of students like student name, age, address, id etc. One way of doing this would be creating a different variable for each attribute. However when you need to store the data of multiple students then in that case, you would need to create these several variables again for each student. This is such a big headache to store data in this way.
We can solve this problem easily by using a structure. We can create a structure that has members for name, age, address and id and then we can create the variables of this structure for each student.

Debugging

Testing and debugging ensure that a program executes successfully for a well-defined range of values. Such a program might still crash for values outside this range. Each program needs to be throughly tested before release to a user community and with each patch to that release. Compilers identify syntactic errors with respect to the rules of the programming language, but cannot readily identify semantic errors; that is, errors in the meaning or intent of the code. Walkthroughs and code analysis help identify these errors. During the development cycle (which includes design, implementation, and test and rework) expect to spend roughly 40% of your time testing, debugging, and reworking your code.
