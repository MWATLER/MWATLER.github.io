Loops

Almost all the programming languages provide a concept called loop, which helps in executing one or more statements up to a desired number of times. In software engineering, a loop is a programming structure that repeats a sequence of instructions until a specific condition is met. Programmers use loops to cycle through values, add sums of numbers, repeat functions, and many other things. Loops are supported by all modern programming languages, though their implementations and syntax may differ. The C language supports three types of iteration: while, do-while, and for loops. Three instructions control the execution of an iteration: an initialization, a test condition, and a change statement. The test condition may be simple or complex.

Arrays

Almost all the programming languages provide a concept called array. An array can store a fixed-size collection of elements of the same data type. An array is used to store a collection of data, but it is often more useful to think of an array as a collection of variables of the same type. Arrays are "lists" of related values. Every value in the array is of the exact same type and only differentiated by the position in the array. For example, all the quiz scores for a test could be stored in an array with the single variable name: quiz_scores. The first student's quiz score would be found in the first "bucket" of the array, the second students score in the "second" bucket, etc.

  • Parallel arrays are arrays that are connected in some way and are used together. Code that uses parallel arrays to track the price and scan-code of items in a grocery store can be found at ParallelArrays.c.

Whether you code in C/C++, JavaScript, Python, Java, PHP, Ruby, etc.. to access the first element of the array you will need to refer to array[0]. This is often confusing for new developers — why do we start numeration from zero? Wouldn’t array[1] be a more obvious choice for the first item? Zero-based numbering comes from language design itself. In C an array points to the location in the memory, so in the expression array[n], n should not be treated as an index, but as an offset from the array’s head. Referencing memory by an address and an offset is represented directly in computer hardware on virtually all computer architectures, so this design detail in C makes compilation easier.
