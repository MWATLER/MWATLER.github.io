Electronic Circuit

  • In week 6 we looked an an electronic circuit. There were inductors, capacitors and resistors. Our code passed current through these electronic components and calculated the voltage across each. Now we are going to do the same thing, but we are going to introduce a base class for each of these components. The base class or parent class will be called Component. The derived classes or children classes are Resistor, Capacitor, and Inductor. The main() function will access these derived classes through their base class, allowing all objects of all three kinds to be treated similarly.
    The UML diagram can be seen at:
    SeriesCircuit.PNG, The code can be seen at:
    Component.h, Resistor.h, Resistor.cpp, Capacitor.h, Capacitor.cpp, Inductor.h, Inductor.cpp, and SeriesCircuit.cpp
  • In order to demonstrate the C++ interface, an interface class has been introduced, which acts as the parent class to Component which in turn acts as the parent class to Resistor, Inductor and Capacitor. The interface class is accompanied by a CreateComponent function. The idea is, the user only sees the interface without knowing how the electronic components are built. The CreateComponent function will act as a user-interface to guide the user in building the electronic circuit.
    The UML diagram can be seen at:
    iSeriesCircuit.PNG, The code can be seen at:
    iComponent.h, Component.h, Component.cpp, Resistor.h, Resistor.cpp, Capacitor.h, Capacitor.cpp, Inductor.h, Inductor.cpp, and SeriesCircuit.cpp
