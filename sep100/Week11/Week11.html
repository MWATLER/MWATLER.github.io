Air Force

  • We want to write a program that simulates travel for many planes in an airforce. Each plane has a list of the pilot's names, the amount of fuel in the tank, the fuel capacity which is fixed at 3000L, and the fuel efficiency which is fixed at 2km/L.
  • Each plane can be created by providing a list of pilot's names and the initial amount of fuel in the tank. Each plane can also be created with no information, but the pilot's names can be added later, as well as the amount of fuel in the tank. The plane should travel a given number of kilometres, and indicate whether such a distance is possible. The plane should print out a report of the pilot's names and how much fuel is left in the tank. When the plane object is destroyed, it should clean up any dynamic memory allocated.
  • The main function is going to have an unknown number of planes. Since the number of planes is unknown at compile time, we will have to dynamically allocate the planes at runtime. Also, the main function will prompt the user for the number of pilots for each plane and the name of each pilot. The main function will travel each plane, then give a report. Since the planes were dynamically allocated at runtime, they have to be deallocated before the main function exits.
  • We clearly have a plane class with related variables and functions which act on them. Since the number of pilots is not known at compile time, we will have to dynamically allocate these names at runtime. We will need an array of names, as well as the size of the array.
  • When travelling, we have to get the range of the aircraft and compare it to the distance we want to travel. If the aircraft has the range, then we will have to calculate the amount of fuel consumed and subtract it from the fuel we have.
  • A solution for the air force can be seen at: Plane.h, Plane.cpp, and AirForceMain.cpp.
