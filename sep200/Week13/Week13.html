A Network Node

Threads are often used in servers or networking software. One thread receives incoming messages, another thread processes these messages and sends them on to their destination. One such example of a network node is given. Mutexes are emulated with mutex initialize, lock and unlock functions in order to protect a shared resource. The messages received in the receive thread are put in a queue. These messages are popped in the send thread and forwarded on to their destination. The input data is simulated via text files (Input1.dat...Input10.dat) and the destination for these messages are simulated by writing the data to an output file (Output.dat). All files are as follows:
NetworkNode.h, NetworkNode.cpp and NetworkNodeMain.cpp.
The input data files can be found at: Input1.dat, Input2.dat, Input3.dat, Input4.dat, Input5.dat, Input6.dat, Input7.dat, Input8.dat, Input9.dat, Input10.dat.
The output data file can be found at: Output.dat.

Thread Race Conditions

Sometimes a function has to wait for a thread to run before processing data. It does take a little time for a thread to start. Synchronization can be obtained with the future, packaged task or async. Programs that demonstrate these are:
ThreadFuture.cpp, ThreadPackagedTask.cpp and ThreadAsync.cpp.

C++ Language Standards

  • The C++ language is evolving over time. See C++ Standards for a pictoral summary. A more detailed document can be found at History of C++.
  • There are many libraries that extend the functionality of C++. For example, Blackberry uses the the Adaptive Communication Environment (ACE) library for concurrent communication software.
  • The Boost C++ Libraries offers extensions to every part of the C++ library. See the documentation of their latest release Boost 1.75.0 Library Documentation for more information.
  • For a full list of C++ compatible libraries see cppreference - A list of open source C++ libraries. When you work in a specialized environment, these libraries become indispensible.
