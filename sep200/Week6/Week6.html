Math Test

The program MathTest shows how to perform a square on a number and a square on a fraction. It does not matter what type the numbers are. They could be int, float, double, etc... The code demonstrates the use of function templates:
Math.h and MathTest.cpp.

A Database

A generic database has been created to perform functions on many data schematics. The database can find a data element by name, get the number of data elements, get a data element by index, get the last data element, and get the first data element. The data element can be any kind of structure but must have a variable name which acts as the key into the database.
An example is created with two types of data schematics (Database2, Database2).
The code for the database can be found in Database.h and the test code for the data base can be found at DatabaseMain.cpp.

Math Test with Polymorphism

A second MathTest program has been created to show how to implement inheritance with templates. This math library performs calculation on a series of numbers. The calculations are to add the series, or to multiply the series. The numbers could be of any numerical type. The base class exposes two virtual functions: AddSeries() and MultiplySeries(). There is a MathAdd derived class for adding a series of numbers and there is a MathMultiply function for multiplying a series of numbers. Templated are the type of numbers to be operated on, as well as the length of the series itself.
The code is as follows: MathTest2.h and MathTest2.cpp.

Software Design with Object Oriented Programming based on the Use-Case Scenario

We had an introduction to software design in SEP101 - Programming Fundamentals. We now want to extend the concepts taught there towards object oriented programming.

  • This requires breaking a problem down into classes, and breaking classes down into a collection of variables (nouns/adjectives) and functions (verbs) which act on those variables.
  • Furthermore, it requires collecting common items into a base class, and specialized items into derived classes.
  • If variables and functions require a variety of data types, they need to be templated, regardless of whether they belong in a class or not.
  • With polymorphism, this allows access to all derived classes through a common base class, where all functions exposed in the base class are declared as virtual.

The Community Library

Let us look at our Community Library once more. In this example, we see all library resources have a title, a catalog ID, a check-out date and a check-in date. These nouns/adjectives can all be collected in a base class as variables: title (string), catalogID (int), checkOutDate (time_t), and checkInDate (time_t). Fees need to be calculated for these resources. We also need to get and set these values. The actions performed on them can be collected as functions in the base class, and overriden in derived classes: LibraryResource(int ID) constructor, CheckOut(time_t), CheckIn(time_t), double CalculateFees(), string GetTitle(), and int GetCatalogID().

The calculation of fees depends on the type of library resource. The library resource could be a standard book, an electronic book, a DVD, a musical instrument, or other resources (the default case). Other resources will rely on the CalculateFees() function in the base class, but the other resources have to override the CalculateFees() function. The details for each can be seen in the document.

In addition to library resources there is a patron. This requires only one class because there is only one type of patron. From the document it is seen that a patron has an id, a full name, a list of library resources and a record of late fees. These can be stored as variables in the patron class. The patron can borrow resources and return resources. A report of the resources can be generated and any fees calculated. Functions can implement all these actions in the patron class.
