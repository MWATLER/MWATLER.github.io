Introduction

This week will be an introduction to object oriented programming. We will also look at compilers and linkage, as well as some fundamental types. We will look at static variables and functions, as well as external variables. During our lab period, we will look at the preprocessor which performs preliminary operations on C and C++ files before they are passed to the compiler. The preprocessor can assist us with debugging our code.

The Community Library

This example shows most of the principles of object oriented programming that we will learn in the first half of the course. The software to implement the community library represents a real-world situation that is implemented using object-oriented programming.

  • A community library is described in CommunityLibraryDescription.docx. This is based on the lending policy of the Toronto Public Library, with exceptions. This library has patrons (customers) who borrow its resources, which could be books, DVDs, musical instruments etc...
  • A document which breaks down the components of the community library can be seen in the UML diagram CommunityLibraryUML.docx. This document shows the relationships between the library resource and its component parts (eBook, standard book, DVD, musical instruments) as well as the patron which can borrow up to 5 library resources at a time.
  • The source code for the library resource can be found at LibraryResource.h and LibraryResource.cpp.
  • The source code for the book can be found at Book.h and Book.cpp. The source code for the eBook and the standard paper book can be found at ElectronicBook.h and ElectronicBook.cpp, StandardPaperBook.h and StandardPaperBook.cpp.
  • The source code for the DVD can be found at DVD.h and DVD.cpp.
  • The source code for the musical instrument can be found at MusicalInstrument.h and MusicalInstrumentDVD.cpp.
  • The source code for the patron (customer) can be found at Patron.h and Patron.cpp.
  • The main function for the community library can be found at CommunityLibrary.cpp where two patrons are created, borrow library resources, then return them three days later. A report for each patron is generated and printed to the screen.

Employee Information

The example employee information shows the use of static class variables to keep track of how many objects have been created as well as a running total. The employee information class contains variables for an employee's name, position and age. The static variable numEmployees keeps track of the number of objects created of type EmployeeInfo. One of the functions of EmployeeInfo returns the average age. This function makes use of the static variable runningTotal where the average age is the running total of ages divided by the number of employees. It is very common to employ a static variable to keep track of the number of objects created from a given class.
The example shown also demonstrates the use of enumerations. A common application for enumerations is the error status returned by functions. See:
EmployeeInfo.h, EmployeeInfo.cpp and EmployeeInfoMain.cpp.

Unions

The members of a union are arranged in parallel in memory. Unlike the members in a class or a struct, the members in a union share the same address in memory An object of union type can only hold the value of one of its members at any particular time. An example that shows how unions are used in conjunction with enumerations to resolve which variable in a unions is to be accessed can be found in Product.cpp.

Data Stream

Often streams of data come with a fixed width - 8 bits, 16 bits, 32 bits,... C++ has types with specific bit sizes which are used for parsing data streams. The code dataMain.cpp demonstrates the parsing of a 16 bit stream.

Generic Parameter

The data type (void *) represents no data type. Parameters can be passed to a function as (void *) and cast in that function to whatever type we wish. This is often used with threads where a thread is represented by a function that runs on its own and accepts one paramater of type (void *). A parameter of any type can be passed through to the thread. If multiple parameters need to be passed through to the thread, they can be packages into one structure and the structure passed through. The code GenericMain.cpp demonstrates the passing of a parameter by value or address, as well as the passing of a structure by value or address.

Macros

Macros in C++ have a wide variety of application. One such application is in debugging. Embedded debug can be enabled and disabled via a simple define. Code with embedded debugging can also apply levels or severity of the debug. Debug logs can be filtered according to their severity. Code with embedded debug can be seen at:
Automobile.h, Automobile.cpp and TravelSimulator.cpp.

With this second example, the embedded debug has been enhanced in this code example to include an enumeration of log levels (LOG_INFO, LOG_WARNING, LOG_ERROR, LOG_CRITICAL) as well as a nested-if inside the Log() macro:
Automobile.h, Automobile.cpp and TravelSimulator.cpp.
