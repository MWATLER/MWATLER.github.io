A Data Acquisition System

I have expanded on the radio channels code of week 5b to turn it into a data acquisition system. Sixteen clients try to communicate with a radio through four radio channels via shared memory. Another 16 clients attempt to communicate with the radio via IP. The radio collates all this information and transmits it to a central data acquisition system which dumps everything to a file.

  • The radio application and the first 16 clients can be found at: client.h, client.cpp, radioApplication.cpp and Makefile.
    The memory map can be seen at: RadioChannelsMemoryMap.docx.
    Scripts for starting and stopping the simulation can be found at: start.sh and stop.sh.
    Each radio channel uses a part of shared memory. The clients write to one of the channels by writing into the channel's shared memory. The use of semaphores ensures that the number of clients writing to the radio's channels is limited to four.
  • The code for the IP clients can be found at: clientIP.cpp, clientRogue.cpp and Makefile.
    Scripts for starting and stopping the simulation can be found at: start.sh and stop.sh.
  • Code for the data acquisition system can be found at: dataAcquisition.cpp and Makefile.
    A sample data file looks like dataFile.txt.
